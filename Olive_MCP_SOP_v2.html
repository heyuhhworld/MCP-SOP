# 工具设计说明书 · SC-001 · 客户持仓查询

## 一、工具拆解与调用说明
工具名: 产品信息查询
回答问题: 产品基本信息、净值、业绩、费用、持仓标的是什么？

预计到期日   | 封闭式到期日；开放式为null    | iGIMS
是否已确认   | true=已公告；false=预计变更  | iGOWIN | 内部

# Tool: get_client_holdings
scene: SC-001 | version: v1.0

## 功能
查询客户在 Olive 所有产品的持仓汇总：
认缴额、已出资、当前市值、累计分配。

## 什么时候调用
用户问：我投了多少钱 / 现在值多少

## 什么时候不调用
询问产品业绩 → 用 get_product_fact
询问行业分布 → 用 get_exposure

## 输入
client_id: string（必填）
as_of_date: date（选填，默认今日）

## 输出示例
{ "total_commitment": 10000000,
  "total_nav": 7200000,
  "total_distributed": 800000 }

## 权限
PII: 无
批量: 最多50条持仓记录
上线前验证
业务验证
定义的场景下，终端用户拿到的最终结果是否符合业务预期
▶
  • 投资场景——用真实产品id调工具，AI 生成的 IC 备忘录结论是否准确，与人工版本对比
  • 客户场景（IR / RM）——用真实 client_id，验证持仓汇总 / 流动性时间轴 / 收益归因的分析结论，是否回答了 P1 中定义的用户问题
  • 合规场景——穿透数据展示是否符合 disclosure_level 规则；非专业投资者 holding_name 是否正确匿名

接口验证
MCP 传递的数据本身是否正确——独立于业务场景判断
▶
  • 各字段返回值与来源系统（iGIMS / iGOWIN / TA）数据一致，无截断、无丢失
  • 数字单位、精度、口径与字段说明书一致（如 DPI 是小数还是百分数）
  • 权限控制正确：disclosure_level 按客户类型自动应用，越权请求返回 403
  • 错误码 / 降级逻辑正常（字段为空时返回 null 而非报错）
上线后持续运作

✅ 业务的日常运营职责
  • 变更发起——任何字段变更 / 新增场景，必须由业务书面提需求，不得口头告知科技修改
  • Bad Case 收集——日常发现 AI 回答有误，记录并提交复盘（每月至少一次正式复盘）
  • 新场景推进——持续补充场景定义表，将 P1/P2 场景逐步纳入开发排期
  • 紧急情况响应——AI 生成内容出现实质性错误，立即停止使用并通知科技

# 变更处理流程（字段/口径/权限/新增场景）

01 业务书面发起
- 提交：变更说明、影响范围、期望上线时间、回归用例

02 科技评估与实施
- 评估：数据/工具/下游Agent/报表影响
- 实施：开发、自测、变更记录

03 合规/权限校验
- 若涉及披露/权限字段：对照审批文件复核

04 双方确认 → 验收 → 上线
- 验收：关键问题回归 & 权限回归
上线 & 持续监控
运行保障：监控、告警、权限审计、故障响应与发布支持
▶
  • 监控与告警——工具可用性、延迟、错误率、关键依赖（DB/API）健康度
  • 权限审计——权限规则变更留痕；定期抽查 disclosure_level / PII 截断是否符合审批
  • 故障响应——异常快速定位、临时降级/熔断、回滚与复盘（含业务通知机制）
  • 发布与变更支持——按变更流程执行灰度/窗口期上线，确保回归与验收闭环
